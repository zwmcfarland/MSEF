1 # A PHP login script (MINIMAL VERSION)
2 
3 A simple, but secure PHP login script. Uses the ultra-modern & future-proof PHP 5.5.
4 BLOWFISH hashing/salting functions (includes the official PHP 5.3 & PHP 5.4 compatibility pack, which makes those
5 functions available in those versions too). Follow the project on **[Twitter](https://twitter.com/simplephplogin)**,
6 **[Facebook](https://www.facebook.com/pages/PHP-Login-Script/461306677235868)** or
7 **[Google+](https://plus.google.com/104110071861201951660)** and have a look on the official support blog
8 **[Dev Metal](http://www.dev-metal.com)**. Ask questions in the [Official Support Forum](http://support-forum.php-login.net/).
9 
10 **This script is part of the php-login project, a collection of 4 different login scripts**.
11 See [php-login.net](http://www.php-login.net) for more info.

17 1. **One-file version:** Full login script in one file. Uses a one-file SQLite database (no MySQL needed) and PDO.
18  Features: Register, login, logout.
19  https://github.com/panique/php-login-one-file
20 2. **Minimal version** All the basic functions in a clean file structure, uses MySQL and mysqli.
21  Register, login, logout.
22  https://github.com/panique/php-login-minimal
23 3. **Advanced version** Similar to the minimal version, but full of features.
24  Uses PDO, Captchas, mail sending via SMTP and much more.
25  https://github.com/panique/php-login-advanced
26 3. **Professional version** Everything comes with a professional MVC framework structure, perfect for building
27  real applications. Additional features like: URL rewriting, professional usage of controllers and actions, PDO, MySQL,
28  mail sending via PHPMailer (SMTP or PHP's mail() function/linux sendmail), user profile pages, public user profiles,
29  gravatars and local avatars, account upgrade/downgrade etc., login via Facebook, Composer integration, etc.
30  https://github.com/panique/php-login

36 ## Requirements
37 
38 - PHP 5.3.7+
39 - MySQL 5 database (please use a modern version of MySQL (5.5, 5.6, 5.7) as very old versions have a exotic bug that
40 [makes PDO injections possible](http://stackoverflow.com/q/134099/1114320).
41 - activated mysqli (last letter is an "i") extension (activated by default on most server setups)
42 
43 ## Installation (quick setup)
44 
45 Create a database *login* and the table *users* via the SQL statements in the `_install` folder.
46 Change mySQL database user and password in `config/db.php` (*DB_USER* and *DB_PASS*).

53 ## Security notice
54 
55 This script comes with a handy .htaccess in the views folder that denies direct access to the files within the folder
56 (so that people cannot render the views directly). However, these .htaccess files only work if you have set
57 `AllowOverride` to `All` in your apache vhost configs. There are lots of tutorials on the web on how to do this.
